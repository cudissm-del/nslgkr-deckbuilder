# 삼국지: 천하결전 — 전투 메커니즘 통합 정리

> 이 문서는 `docs/01_전투매커니즘_분석.md`(구조·아키텍처)와 `docs/02_전투흐름_정리.md`(계산 순서),
> `data/combat.json`·`data/arms.json`·`data/battles.json`의 확정 수치를 **하나로 합친 마스터 레퍼런스**다.
> 덱빌더의 전투 계산은 이 문서를 단일 진실원(single source of truth)으로 삼는다.
>
> 최종 통합: 2026-07-21 · 원본 문서는 이력 보존용으로 유지

## 0. 한 줄 결론

**전투는 서버가 자동 계산한다. 조작 개입이 없으므로 승패는 부딪히기 전 세팅에서 사실상 결정된다 — 덱 최적화가 곧 전투력이다.**

---

## 1. 아키텍처 (설치 파일·로그로 확인)

- **엔진/배급**: Unity PC 클라이언트(`com.bhk.newslgkr`), 빌리빌리(biligames) 계열 한국 서버.
- **서버 권위형(server-authoritative)**: 전투는 항상 서버 응답으로 들어온다.
  - `RPCGetWarBattleViewResponse` — 전투 재생 데이터 요청
  - `OnShowBattleReport` / `BattleDetailUI show battleID:...` — 전투마다 고유 `battleID`의 보고서로 저장·조회
  - → **계산 로직과 마스터 수치는 클라이언트가 아니라 서버에 있다.**
- **로컬 DB**: `nslgLiteDb_*.db`는 SQLite가 아니라 **LiteDB(.NET NoSQL)**. 컬렉션은 `hero`·`item`·`skill`·`worldHornIsUseItem`뿐이고 전부 **숫자 ID**로만 저장 — 상성표·전법 계수 같은 마스터 수치는 못 뽑는다.
- **에셋 번들**: `.ns` 확장자, **UnityCN 암호화 + NEP2 안티치트**. 게임 로직은 암호화된 Lua 패치(`lua-patches/GenesisPatch`) 안. 평문 추출 비효율적.
- **로그에서 확인된 시스템 모듈**: `army`(병력) · `arms`(병종, 승급 존재) · `skill`/`tactics` · `battlebuff` · `tower`(원정) · `campwar`(연맹전) · `thiefbattle` · `arena` · `aiarmy` · `replay` · `policy`(`policyDesignatedArmsBuff` = 지정 병종 버프)
- **병종 승급**: `RPCGetAllUnlockedArmsAdvancedInfoResponse` 등 — 병종에 해금/승급 단계가 존재.

> **실질적 데이터 수집 경로**: ① 화면 녹화 → 프레임 추출 → 판독, ② PDF 저장 → PyMuPDF 렌더 판독, ③ 텍스트 복사(F12 `copy()` 우회). 번들 복호화는 후순위.

## 2. 전투의 기본 단위

| 항목 | 내용 |
|---|---|
| 부대 | 장수 **3명** |
| 병력 | 장수마다 개별 병력 풀. **상한 = 무장 레벨 + 병영 레벨** (병영 5→6: 500→700) |
| 궤멸 | 장수 병력 0 → 궤멸. 한쪽 부대 전멸 → 전투 종료 |
| 장수 구성 | 병종(방패/궁/창/기마) · 위치(전열/후열) · 스탯 4종 · **전법 3**(고유1+장착2) · **병법 3** · 장비 |
| 보상 | 승리 시 무훈(武勳) 지급, 손실·공급 상태 함께 표시 |

### 스탯 4종

| 스탯 | 역할 |
|---|---|
| **무력** | 주는 **병기** 피해 |
| **지력** | 주는 회복 + 주고받는 **책략** 피해 |
| **통솔** | 받는 병기/책략 피해 (**순수 방어 스탯**) |
| **선공** | 매 턴 행동 순서 (높을수록 먼저) |

**속성치 분배**: 10레벨마다 +10, 승급마다 +10, 리셋 가능.
추천: 병기→무력, 방어→통솔, 책략/치유/보조→지력, 문무→무력+선공.
예외: 견희는 지력형이나 전열 배치가 많아 통솔 투자 유효. 디버프형은 선공 투자 유효.

## 3. 전투 계산 파이프라인 (핵심)

### 3-0. 편성 단계에서 확정되는 보정

**진영 보너스**

| 동일 진영 | 효과 |
|---|---|
| 2명 | 모든 속성 +5% |
| 3명 | 모든 속성 +10% |

**병종 편성 보너스**

| 병종 | 2명 | 3명 |
|---|---|---|
| 방패병 | 받피 −3.5% | 받피 −5% |
| 창병 | 주피 +2.1% / 받피 −1.4% | 주피 +3% / 받피 −2% |
| 궁병 | 주피 +3.5% | 주피 +5% |
| 기병 | 주피 +1.4% / 받피 −2.1% | 주피 +2% / 받피 −3% |

**인연(羈絆)** — 특정 무장 조합 동시 출전 시. 예) 천하삼분 = 조조+유비+손권 → 인연 무장 받피 −6%(호위의 의지)

**진형** — 안형진: 주피 +15%, 통솔 +20 / 기형진: 받피 −6% / 방원진·일자진 등. 내성 연구소에서 신규 진형 해제.

---

### 3-1. 포진(셋업) — 로그상 실제 누적 순서

```
1. 공급 판정        공급 100 만점. <80 → 주피 −25% / <40 → 주피 −50%
2. 진형 효과
3. 병법(병서) 효과   연사 통솔+15 · 병정 받피−5.34% · 기동 액티브발동률+3%
4. 편성/진영 보너스   ※ 로그엔 대상 진영별로 분해되어 표시
                    ("오국 무장에게 주는 피해 +2%", "촉국 무장에게 받는 피해 −2.5%")
5. 지휘 전법 발동     전법 중 가장 먼저, 효과 지속(오라)
        ↓
   전투 중 실제 스탯값 확정
   (로그 실측: 주유 지력 362.50 / 통솔 249.24 / 선공 174.55, 소교 지력 302.48, 대교 통솔 252.14)
```

### 3-2. 행동 순서 결정

**선공(先攻)** 수치로 매 턴 순서 결정. 로그에 "행동 순서 판단 완료" 표시.

> **선공은 속도가 아니라 주도권이다.** 디버프/버프 담당 장수의 선공을 최우선으로 올리는 이유 = 적의 침묵·공포를 맞기 전에 내 전법을 먼저 꽂기 위해. (예: 소교 선공 최고 → 적 침묵 전에 허약 부여)

### 3-3. 각 장수의 턴

```
액티브 전법  (일반 공격 前, 슬롯 위→아래 순서, 각자 발동률% 판정)
      ↓
   일반 공격
      ↓
추격 전법    (일반 공격 後)
```

- **패시브**는 조건 충족 시 별도 발동 (전투 시작 / 턴 시작 / 조건부)
- **지휘**는 이 루프 이전, 셋업 단계에서 발동
- **슬롯 순서 = 발동 순서 → 콤보 설계 가능**: 앞 슬롯에서 상태 부여 → 뒤 슬롯에서 그 상태 활용 (예: 화공전술 → 재해이용)

**전법 특성 ↔ 스탯 매칭**

| 특성 | 짝 스탯 |
|---|---|
| 병기 | 무력 |
| 책략 | 지력 |
| 치유 · 보조 | 지력(아무 무장 가능) |
| 방어 | 통솔(방어력 높은 무장) |
| 문무 | 무력 + 지력 양쪽 |

### 3-4. 피해 계산

로그 표기: **`병력이 A(B) 손실`** → **B = 기본(경감 전) 피해, A = 실제(경감 후) 손실**. 경감률 = 1 − A/B.

```
[기본 피해 B]
  = 전법 계수 × 스탯(병기→무력 / 책략→지력)
  × 증폭: 회심(병기 ×1.5) · 묘책(책략 ×1.5)
        · 진형 / 병법 / 편성 버프
        · 병종 상성 우위
        ↓
[경감]
  목표 통솔 · 병종 상성 열세 · 받는 피해 감소 버프
  · 방어 스택(1회 소모 시 −71% 관측, 규칙상 70~90%)
  · 허약(주는 최종 피해 −70%) 등 디버프
        ↓
[실제 손실 A]
```

**실측 사례** (battles.json b002 / 전황 정보 탭 로그)

| 공격 | 기본 B | 실제 A | 경감률 | 비고 |
|---|---|---|---|---|
| 주유 → 소교 (기지의 승리) | 679 | 402 | 40.8% | 일반 |
| 주유 → 주유 (기지의 승리) | 3,459 | 503 | 85.5% | 묘책(×1.5) 발동 + 방어스택(−71%) 소모 |

### 3-5. 부대 단위 페널티 (곱연산)

| 조건 | 효과 |
|---|---|
| 공급 < 80 | 주는 피해 −25% |
| 공급 < 40 | 주는 피해 −50% |
| 부대 병력 < 최대 50% | 주는 피해 −20%+ |
| 전투력 낮은 수비군 상대 | 체력 소모 절반, 획득 수익 −20% |

**공급 회복/감소**: 범위 내 30초당 +1 / 범위 밖 60초당 −1.
**범위**: 주성 주위 5칸 / 동맹 공성·보급 주둔지 주위 12칸 / 동맹 성지.

> 병력 50% 페널티 때문에 **한 번 밀리면 가속되는 구조**다. 교환비가 무너지기 시작하면 되돌리기 어렵다.

## 4. 병종 상성

**방패 → 궁 → 창 → 기마 → (다시) 방패** (화살표 = "제압", 시계방향 4중 순환)

| 병종 | 제압 | 약함 | 중립 |
|---|---|---|---|
| 방패병 | 궁병 | 기병 | 창병 |
| 궁병 | 창병 | 방패병 | 기병 |
| 창병 | 기병 | 궁병 | 방패병 |
| 기병 | 방패병 | 창병 | 궁병 |

- 2칸 건너(정반대) 병종과는 **상성 없음(중립, ×1.0)**.
- ⚠️ **제압/피제압 배수(%)는 미확보** — 현재 프로젝트 최대 구멍.
- 웹 공략의 "보병 > 궁병 > 기병"(3종)은 **단순화 오류**. 실제는 방패병/창병이 분리된 4종 순환.
- 지도 확대 시 부대 모양으로 병종 식별 가능(방패/창/활/말).
- 정책(`policyDesignatedArmsBuff`)·병종 승급으로 특정 병종 강화 → **"주력 병종 + 그 병종 강화 정책/승급"의 정렬**이 중요.

## 6. 종료와 후처리

- 장수 병력 0 → **궤멸**. 한쪽 부대 전멸 → 전투 종료.
- **중상**: 부대 패배 시 전 무장 중상 → **5분간 출정 불가**(초보 1분). 청낭 직업의 *응급처치약*(예비병 반환+시간 단축)·*금창약*(즉시 해제)으로 완화.
- **휴군**: [파죽지세] 단계 해제 후 활성. 유저 부대와 전투 시 무장 중상마다 **3스택**(10분 지속, 최대 20). 휴군 스택 1개당 중상 시간 +1분.
- **전황**: 전투 후 승패·상세 과정 자동 기록. 출전 전 토지를 클릭해 **수비군 조합 사전 파악** 가능.

## 7. 장수 강화 3축

```
장비 (기본 스탯)  →  병서 (상시 보정)  →  전법 (실전 발동)
        └──────── 곱연산으로 누적 ────────┘
```

| 축 | 역할 | 해금 |
|---|---|---|
| **전법(戰法)** | 부대의 딜·버프·힐·제어 대부분이 여기서 나옴. 고유 1 + 전수받은 2. 레벨업 시 계수·발동률 상승 | 슬롯 10·20레벨 |
| **병서(兵法)** | 장수별 패시브 특성. "초반 N턴 피해 증가, 병력 상한 +, 상성 보정" 같은 상시 보정 | 슬롯 3개: 25·30·35레벨 (동전 1만/3만/5만) |
| **장비(裝備)** | 무력·지력·통솔·선공 등 기본 스탯 + 부가 효과. 강화·제련으로 성장. 스탯이 오르면 전법 계수도 함께 커짐 | 15레벨 |

> **덱 최적화 1순위 변수는 전법.** 같은 장수라도 이 3축 세팅에 따라 전투력이 크게 갈린다.

### 병법(병서) 상세

- **등급 4종**: 전설 / 선본 / 수기 / 잔편
- **재연구**: 3병법 모두 해제 후 가능. 책자 4개 소모/회, 1개(3택) 무작위 리셋. 10회 시 전용 병법 확정. 10회 후 잔편 미등장, 20회 후 수기 미등장.
- **효과 예시**: 근선(회복 후 1턴 통솔+10) · 연사(통솔+15) · 질전(4턴간 추격 전법 피해+9%) · 파적(관통+6%) · 무봉(4턴간 일반공격 피해+12.5%)

## 8. 플레이어 직업 6종

| 직업 | 성격 |
|---|---|
| 진군 | 전면 전투·생존 특화 탱커 |
| 신행 | 기동성·PvP 특화 딜러 |
| 천공 | 광역 전투·사냥 효율 |
| 기좌 | 전략 스킬 운용·지원 |
| 청낭 | 회복·유지력 (중상 완화 약 보유) |
| 병참 | 자원 운영·성장 보조 |

직업 스킬과 병종의 **조합 시너지**가 단일 캐릭터 성능보다 중요하다.

## 9. 전투 콘텐츠(모드)

- **원정/타워(tower)** — 층마다 버프를 골라 쌓는 로그라이크식 (`TowerBuffSelectUI`, `curExpeditionBuffPool`)
- **연맹전/캠프전(campwar)** — 대규모 공성·연맹 조합전. **여기서는 개별 장수 성능보다 부대 전체 조합이 승패를 좌우**
- **도적전(thiefbattle)**, **경기장(arena)**, **개척/사냥(aiarmy)**
- 월드맵 도시 공격 예약·행군 (`RPCAllArmyPredetermineAttackCityResponse`)

## 10. 연구소·시즌 보정

- **연구소**: PVP 필수. 진영별 공방 보정 — 예) "오국 저지" 4→5레벨 = 오국 무장에게 받는 피해 2%→2.5% 감소
- **병영 레벨**: 부대 병력 상한 증가 (5→6: 각 무장 500→700)
- **전법 레벨업**: 예) "백성과 함께" 7→8레벨에서 통솔 15→16, 치유율 83.33%→88.89%
- **성지 보강**: [중흥의 주인] 단계 오픈. 본 성지·이전 범위 내 주성 내구도 피해 90% 감소, 동맹 부대 주둔 시 피해 증감 +15%

## 11. 덱빌딩 원칙 (실전 요약)

1. **승패는 전투 전에 결정된다** — 서버 자동 계산, 조작 개입 없음.
2. **보정은 곱연산으로 쌓인다** — 진영·병종·인연·진형·병법·공급이 전부 곱해져 기본 피해와 경감에 반영.
3. **선공 = 주도권** — 디버프 담당 무장의 선공을 최우선으로. 적 침묵·공포보다 먼저 꽂아야 한다.
4. **통솔은 순수 방어, 지력은 회복+책략 양방향** — 딜러는 전법 특성에 맞는 스탯으로 몰빵.
5. **3인을 같은 진영/병종으로** — 동일 진영 3명 +10%, 궁병 3명 주피 +5% 등 편성 보너스 극대화.
6. **액티브 슬롯 순서로 콤보** — 앞 슬롯 상태 부여 → 뒤 슬롯 활용.
7. **유지선 사수** — 병력 50% 이상, 공급 80 이상. 미만 시 주피 대폭 감소.
8. **출전 전 정찰** — 적 토지 '전황'과 부대 병종(방패/창/활/말) 확인해 상성 우위 확보.
