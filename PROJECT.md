## 2. 전투 구조 (한 줄 요약)

전투는 **서버가 자동 계산**한다(실시간 조작 아님). 부대를 행군시켜 충돌하면 서버가 결과를 계산하고 클라이언트는 **전투 보고서를 재생**만 한다. 따라서 승패는 부딪히기 전 세팅으로 사실상 결정 — **덱 최적화가 곧 전투력**.

- **단위**: 1부대 = 장수 3명, 각 장수가 병력 풀을 지휘. 병력 0 → 궤멸.
- **자동 전투**: 장수 전법이 라운드마다 발동. 한쪽 전멸 시 승패 결정, 무훈 지급.
- 상세: `docs/01_전투매커니즘_분석.md` 참조.

## 3. 확정된 핵심 메커니즘 (`data/combat.json`)

**스탯 역할**
- 무력 = 주는 병기 피해 / 지력 = 회복+책략(주고받음) / 통솔 = 받는 피해(방어) / 선공 = 행동 순서

**병종 상성 (4종 순환)** — `방패 > 궁 > 창 > 기마 > 방패` (제압 관계, 2칸 건너 중립)

**편성 보너스** (부대 3인 시너지)
| 구성 | 2명 | 3명 |
|---|---|---|
| 동일 진영 | 모든 속성 +5% | +10% |
| 방패병 | 받피 −3.5% | −5% |
| 창병 | +2.1%/−1.4% | +3%/−2% |
| 궁병 | 주피 +3.5% | +5% |
| 기병 | +1.4%/−2.1% | +2%/−3% |

**부대 피해 페널티**
- 공급 <80 → 주피 −25%, <40 → −50% (공급 만점 100)
- 병력 <50% → 주피 −20%+

**전법 유형·타이밍**: 지휘(가장 먼저·지속) → 액티브(공격 전) → 추격(공격 후), 패시브(조건). **액티브는 슬롯 위→아래 순서로 발동 → 콤보 설계 가능.**
**전법 특성↔스탯**: 병기=무력, 책략=지력, 문무=둘 다, 치유/보조=아무나, 방어=방어형.
**상태이상/버프**: 홍수(통솔−20)·폭풍(선공−30)·위협(받피+10%)·허약(최종피해−70%)·침묵·공포·혼란 / 회심·묘책(+50%)·관통·간파·회유·연타·반격·방어(70~90%감소) 등 수치화 완료.
**병법(병서)**: 슬롯 3개(25·30·35레벨), 등급 4종(전설·선본·수기·잔편), 재연구(책자 4개/회).
**속성치 분배**: 10레벨마다 +10, 승급마다 +10. 특성별 추천(병기→무력, 방어→통솔, 책략·치유·보조→지력).

## 5. 데이터 수집 파이프라인 (재현 방법)

1. **영상 녹화** (도감 목록·무장 상세): 천천히 스크롤 녹화 → `captures/videos/`에 저장 → `tools/extract_frames.py`로 프레임 추출 → Claude가 판독·구조화. (도감/상세용)
2. **PDF 저장** (그림 많은 공략글): 브라우저 `Ctrl+P` → PDF로 저장 → Claude가 PyMuPDF로 페이지 렌더 후 판독. (텍스트+다이어그램용, 정확도 최상)
3. **텍스트 붙여넣기** (표 위주 글): 복사 차단 시 F12 콘솔 `copy(...)` 우회. (그림 없는 글용)

> 참고: 게임 에셋 번들은 UnityCN 암호화 + NEP2 안티치트로 보호되어 오프라인 추출은 비효율적. 위 3가지가 실질 경로.

## 6. 남은 과제 (우선순위)

1. **무장 스탯 보강**: 나머지 55명 상세(스탯·병종·전법) — 특히 주력·미획득 핵심 장수.
2. **전법 최고레벨 계수**: 상세 미확보 전법의 효과문·발동률, 전 전법 '최고 레벨' 탭 수치.
3. **병종 상성 배수**: 제압/피제압 정확 % (전투 보고서 역산 또는 상세 툴팁).
4. **장비·병법 데이터**: 스탯·효과 수집(스키마만 존재).
5. **덱빌더 사이트**: 위 데이터를 소비하는 정적 웹앱(GitHub Pages).
